# オフシーズン更新仕様

## 基本方針

- Excelファイルを正本にしない。
- PDFファイルを正本にしない。
- 正本は `data/men/` と `data/ladies/` のCSVとする。
- Excel/PDFはGitHub Actionsで生成する。
- CSV更新はGitHub Actionsでは実行せず、利用者が1日1回Codexにプロンプトを投げ、Codexが公式サイトを確認してCSVを更新する。
- 更新済みCSVのpushをトリガーに、GitHub ActionsでExcel/PDFを生成する。
- AndroidスマホだけでCodexへのCSV更新依頼、CSV確認、Actions実行、Artifact取得まで運用する。
- Codex上ではExcel/PDFを生成しない。
- 既存xlsxは参考扱いであり、今後は正本として扱わない。

## CSV仕様

### players.csv

対象は `data/men/players.csv` と `data/ladies/players.csv`。ヘッダーは必ず以下にする。

```csv
背番号,ポジション,選手名,その他備考
```

### official_events.csv

対象は `data/men/official_events.csv` と `data/ladies/official_events.csv`。ヘッダーは必ず以下にする。

```csv
選手名,動向,発表日,区分,移籍先元,ステータス,出典URL,その他備考
```

女子データは公式サイト確認ベースで反映する。非公式情報や推測は入れない。個別記事URLが確認できる場合はNEWS一覧ではなく個別記事URLを出典URLに入れる。公式サイトで確認できない情報は空欄にし、公式発表が見つからない選手はスマホ表示シート上で未発表扱いにする。

スマホ表示シートの母集団は `players.csv` とする。`official_events.csv` は動向データであり、母集団ではない。`players.csv` に存在する選手は、`official_events.csv` に動向がなくてもスマホ表示シートに出力する。`official_events.csv` に動向がない場合は未発表として扱う。

背番号は `players.csv` を正本とする。スマホ表示シートA列には `players.csv` の背番号を出力する。`official_events.csv` には背番号を持たせない。公式サイトで背番号変更が確認できた場合は `players.csv` の背番号を更新する。背番号不明・未発表の場合は空欄を許容する。推測や非公式情報で背番号を補完しない。背番号不明時は `未確認` と入力せず、空欄にする。`players.csv` はヘッダー行を除き、背番号が数値で入っている選手を数値昇順に並べ、背番号空欄の選手は背番号あり選手の後ろに置く。背番号空欄の選手同士は既存の相対順をできるだけ維持する。


## 移籍・期限付き移籍の区分定義

`official_events.csv` は公式発表ごとの動向イベントであり、スマホ表示シートの母集団ではない。移籍・期限付き移籍はアルビレックス新潟／アルビレックス新潟レディースを基準に、入ってくる動きを `in`、出ていく動きを `out` として管理する。

- `移籍in`: 他クラブから完全移籍で入ってくる動き。`加入`、旧表記の `完全移籍加入`、`移籍加入` を統一する。
- `移籍out`: 他クラブへ完全移籍で出ていく動き。`退団`、旧表記の `完全移籍` は、内容に応じてこの区分や `契約満了`、`引退` などの具体区分に整理する。
- `期限付移籍in`: 他クラブから期限付き移籍で入ってくる動き。旧表記の `期限付き移籍加入` を整理する。
- `期限付移籍out`: 他クラブへ期限付き移籍で出ていく動き。旧表記の `期限付き移籍` を整理する。
- `期限付移籍in延長`: 他クラブからの期限付き移籍加入期間を延長し、引き続きアルビに残る動き。残留系として扱う。
- `期限付移籍out延長`: 他クラブへの期限付き移籍期間を延長する動き。新たな退団イベントではないため、退団系ではなくその他系として扱う。
- `期限付移籍out終了`: アルビから他クラブへ期限付き移籍していた選手の期限付き移籍期間が終了する動き。加入・復帰寄りとして扱い、色も加入系にする。公式発表本文に「復帰」とあっても、CSVの `区分` と `動向` では `復帰` を使わず `期限付移籍out終了` に統一する。
- `期限付移籍in終了`: 他クラブからアルビへ期限付き移籍加入していた選手の期限付き移籍期間が終了する動き。退団・離脱系として扱い、色も退団系にする。

`復帰` は区分として廃止し、期限付き移籍先から戻る動きは `期限付移籍out終了` として扱う。`加入` は `移籍in` と意味が重複するため区分として廃止する。`退団` は `移籍out`、`契約満了`、`引退` などと重複するため区分として廃止する。`新卒加入` はオフシーズン動向では扱わないため、区分として設けない。`期限付き移籍延長` や `レンタル移籍延長` のような旧表記は、公式リリース本文からin/outを判断して `期限付移籍in延長` または `期限付移籍out延長` に整理する。`レンタル中` はイベント区分ではなく状態なので、`区分` には使わず `その他備考` に記載する。

移籍系、期限付き移籍系では、移籍元または移籍先を `移籍先元` に入れてよい。ただし印刷対象は `A:F` のため、スマホ表示シート/PDFで見えるように `その他備考` にも必ず記載する。D列の `動向` は `移籍in`、`移籍out`、`期限付移籍in` など短い新区分名にし、期限や詳細説明は原則F列 `その他備考` に入れる。

公式サイトで `移籍in`、`期限付移籍in`、`期限付移籍out終了` が確認できた選手は、`official_events.csv` だけでなく `players.csv` も追加・更新する。背番号は公式サイトで確認できる場合のみ入力し、不明時は空欄にする。

## スマホ表示シート仕様

スマホ表示シートの列は以下にする。

- A列: 背番号
- B列: ポジション
- C列: 選手名
- D列: 動向
- E列: 発表日
- F列: その他備考
- G列: 区分
- H列: 移籍先/元
- I列: ステータス
- J列: 出典URL

印刷対象は `A:F` とする。印刷対象列は「背番号、ポジション、選手名、動向、発表日、その他備考」。`G:J` は管理用であり印刷対象外とする。

公式発表がない選手は以下の表示にする。

- 動向: 未発表
- 発表日: 空欄
- その他備考: `players.csv` のその他備考。ない場合は空欄
- 区分: 未発表
- ステータス: 未発表

同一選手に複数イベントがある場合は、発表日が新しいものを優先し、同日または日付不明ならCSV上で後ろの行を優先する。

## 色分け

スマホ表示シートでは、区分または動向に応じて行を色分けする。

- 契約更新、期限付移籍in延長: 薄い緑（残留系）
- 移籍in、期限付移籍in、期限付移籍out終了: 薄い青（加入・復帰寄り）
- 移籍out、期限付移籍out、期限付移籍in終了、契約満了、引退: 薄い橙（退団・離脱系）
- 期限付移籍out延長: 薄い紫（その他系。退団・離脱系とは別色）
- 未発表: 薄い黄

## GitHub Actions仕様

Artifact名は `albirex-offseason-excel-and-pdf` とする。`Build Excel and PDF` ワークフローは、CSVがpushされたとき、または手動実行されたときに、更新済みCSVを元にExcelファイルとPDFファイルを生成するだけのWorkflowとする。GitHub Actions内で公式サイトを確認してCSVを更新する処理は行わない。

`Build Excel and PDF` ワークフローで以下を行う。

1. Pythonをセットアップする。
2. `openpyxl` をインストールする。
3. LibreOfficeをインストールする。
4. 日本語フォント `fonts-noto-cjk` をインストールする。
5. `scripts/build_workbook.py` を実行して `exports/*.xlsx` を生成する。
6. `scripts/export_pdf.sh` を実行して `exports/*.pdf` を生成する。
7. `exports/*.xlsx` と `exports/*.pdf` をArtifact `albirex-offseason-excel-and-pdf` としてアップロードする。

PDF変換では、PDF用一時Excelを作り、スマホ表示シート以外を削除してからLibreOffice headlessで変換する。管理用シートがPDFに混ざらないようにする。

## 日次公式サイト確認仕様

日次更新Workflowは作成しない。schedule付きの `Daily CSV Update` workflowも作成しない。GitHub Actionsによる日次自動CSV更新は行わない。CSV更新はCodexへの日次プロンプトで行う。利用者が1日1回Codexにプロンプトを投げ、Codexが公式サイトを確認してCSVを更新する運用とする。更新済みCSVをpushすると、`Build Excel and PDF` が実行され、Excel/PDFをArtifactとして出力する。CSV変更がある場合は二重実行を避けるため `workflow_dispatch` は実行しない。CSV変更がない場合のみ、必要に応じて `workflow_dispatch` を1回だけ実行できる。

`scripts/update_official_csv.py`、`scripts/official_sources.py`、`scripts/csv_utils.py` は将来の自動化検討用または補助スクリプトであり、現時点の正式運用ではGitHub Actionsから実行しない。

### 確認対象

- アルビレックス新潟公式サイトのみを確認する。
- アルビレックス新潟レディース公式サイトのみを確認する。
- 非公式サイト、SNS、掲示板、まとめサイト、推測情報は反映しない。
- 公式サイトで確認できない情報は既存CSVの内容を維持する。

### CSV更新方針

`players.csv` はスマホ表示の母集団であり、公式サイトの選手一覧・プロフィールで確認できる選手、背番号、ポジション、選手名を可能な範囲で反映する。背番号が公式サイトで確認できない場合は空欄にし、`未確認` などの文字列は入れない。既存選手は公式サイトで明確な変更が確認できる場合のみ更新し、公式サイト取得不可または解析不可の場合は削除・上書きしない。

`official_events.csv` は動向データであり、公式ニュースで確認できた契約更新、移籍in、移籍out、期限付移籍in、期限付移籍out、期限付移籍in延長、期限付移籍out延長、期限付移籍in終了、期限付移籍out終了、契約満了、引退などの公式発表だけを追加する。同一選手・同一発表日・同一区分・同一動向の行は重複追加しない。複数イベントがある場合の優先順位は既存仕様どおり、発表日が新しいイベントを優先し、発表日が空欄または同一の場合はCSV上で後ろの行を優先する。

### 障害時の挙動

公式サイトへアクセスできない、サイト構造が変わった、または解析できない場合でも、既存CSVを維持する。推測で新規行を追加せず、既存行を削除せず、公式確認できない値で上書きしない。

### 生成物

Excel/PDFは `Build Excel and PDF` ワークフローで生成する。GitHub Actionsは更新済みCSVを入力として扱い、公式サイト確認やCSV更新は行わない。`.xlsx` と `.pdf` はGit管理対象外の生成物であり、正本にはしない。
